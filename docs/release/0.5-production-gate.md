# Notare 0.5 — Production Gate

The checklist that gates a 0.5 **production** release. Policy (QA/release-gate, 2026-07-22):
an RC may ship with *documented* known edge cases; **production may not ship with any open
`GATE: BLOCK` row.** Every row needs evidence per platform (Linux verified on coruscant;
macOS/Windows gated on Abhishek). RC path: `0.5.0-rc.n` (edge cases allowed) → production (none).

Legend: ✅ pass (evidence) · ⚠️ RC-acceptable known edge case · ⛔ BLOCK (must clear before production) ·
▫️ not yet exercised on this platform.

## E. Systemic finding — crash/interrupt resilience for in-flight capture is a 0.5 gate

Three reliability bugs now share a theme: **an in-flight dictation or recording does not survive an
interrupt** (Windows STOP race, STT degradation-without-detection, macOS recording killed by a UI
reload). This is not three point-fixes — it warrants a **small dedicated hardening pass**: recording as a
backend-owned session decoupled from any window/webview and resumable after reload; the stop/finalize
path never losing buffered audio on a transport hiccup; and health/throughput self-monitoring for the
server. Track as an explicit pre-production hardening item.
